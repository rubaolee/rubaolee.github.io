Abstract

Modern database systems are increasingly shaped by domain-specific hardware, yet using such hardware effectively still requires new system abstractions, programming methods, and algorithmic reformulations. This proposed tutorial presents a case study of hardware-accelerated ray-tracing spatial database systems. We explain how Ray Tracing (RT) cores in commodity GPUs, originally designed for graphics rendering, can be repurposed to accelerate key spatial data operations by treating spatial search and refinement tasks as ray-geometry intersection problems.

The proposed tutorial is centered on four recent projects from our group: RT-based spatial join, RT-based spatial indexing, RT-based acceleration of Hausdorff distance computation, and the integration of RT acceleration into Apache SedonaDB. Together, these works provide a coherent path from hardware principles and RT programming, to query operators and index design, to real system deployment.

Open-Source Code

A central goal of this proposal is to show the open-sourced systems behind the ideas discussed in the session. The following codebases provide concrete starting points for experimentation, reproduction, and further system building.

  • RayJoin - RT-based spatial join acceleration for line-segment intersection and point-in-polygon workloads.
  • LibRTS - A spatial indexing library built on ray tracing hardware.
  • X-HD code | X-HD paper PDF - Fast Hausdorff distance computation with ray tracing.
  • SedonaDB GPU spatial integration - Apache SedonaDB integration path for RT/GPU-accelerated spatial processing.

Audience

The proposed tutorial targets database researchers, systems researchers, GPU programmers, practitioners, and students who are interested in building next-generation query engines on modern hardware. Only a basic background in database systems or GPU programming is assumed, and no prior graphics experience is required.

Topics

  • RT programming fundamentals, BVH traversal, and NVIDIA OptiX
  • RT for spatial join, including line-segment intersection and point-in-polygon
  • RT for spatial indexing with point, range, and mutable-data support
  • RT for spatial functions using Hausdorff distance as a case study
  • System integration in Apache SedonaDB
  • Research opportunities beyond current spatial workloads

Schedule

  • Background and RT Programming - 15 minutes
  • Spatial Join Acceleration - 15 minutes
  • Spatial Indexing - 15 minutes
  • Hausdorff Distance Computation - 15 minutes
  • SedonaDB Integration - 15 minutes
  • Discussion and Q&A - 15 minutes

For an extended 3-hour session, each topic can be expanded with deeper technical discussion, code walkthroughs, and more interactive questions.
